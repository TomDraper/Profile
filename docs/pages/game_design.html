Game Design

Games are really the heart of what I want to do. It's where my passion lies and I love the intricacies of developing systems for players as well as the challenges faced in designing new game elements. I have created small games for my own continued edification while working a stable job after university.

Tools I regularly use:

Godot is an open-source games engine, mainly aimed at 2D sprite based games but does include capabilities for 3D. I've recently switched to this being my engine of choice (over Unity) due to its "tool" keyword, a workflow more suited to individuals, and a more intuitive UI system that in my opinion handles multiple resolutions easier.

Unity and Unreal Engine were taught to me at University. The former of which I have continued to expand my learning on including creation of Editor GUI Tools, custom component systems and utilizing the ECS system when that was introduced. It is by far the engine I have the most experience with.

I use Blendr for when I need 3D models, Krita for digital painting and manipulation, Graphics Gale for 2D sprite creation / animation, and Inkscape for vector drawing. These are all free or open source versions of popular software I have used in the past; Photoshop, Illustrator, 3DS Max, Maya etc.

Godot - Space Game

Created mainly as a way to see how I could utilize control nodes to give instant visual feedback to a developer without them having to dig through an objects resources.

Unity - Fifty Fifty

I had an interesting concept for how a text adventure could be created to create multiple and dynamic storylines procedurally. The main bulk of which would be with creating large amounts of characters, places, and events for the player and then depending on what choices the player made having those characters/worlds seemingly 'know' of the players choice by limiting the selection pool of possible events.

Terrain Generation using ECS

A project to teach myself the ECS architecture, includes a terrain generated using perlin noise with controls to allow easy manipulation. All data is loaded into ECS including collision, simple pathfinding, and pawns.
