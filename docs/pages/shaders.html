Shaders

Learning to write shaders and the way you have to change your thinking was one of the most challenging and rewarding things I've done since university. I've iFramed them in from p5 with a brief description of what each is doing.

Tools I regularly use:

P5 Editor support GLSL2.0 which is enough for my personal needs, easy to access, easy to display.

Shader Toy is great for anything shader related and I often use it as reference if nothing else.

Unity / Godot are Game Engines which for the case of viewport shaders I tend to use because it's less complicated than setting up a rig in P5.

Voronoi

Voronoi are used for all sorts of things but I stumbled upon the fact that if you draw each pixels closest points color, you get a cheap and easy diagram. It is important to note though that this in itself is not producing any 'Voronoi points' and as such you cannot tesselate this for paths, however it works as a very cool demonstration and may have uses in things such as overlays for city builder games.

Falling Sand

Click to add sand. Uses a simple cellular automata rule to dictate falling and sideways movement creating a 'realistic' sand falling. At it's core this is what Noita uses to 'simulate every pixel on the screen', using different rulesets for different pixels; and I think that is amazing.
